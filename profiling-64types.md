# 習慣化プロファイリング — 64類型設計

> 3軸 × 4分類 = 64の行動類型
> プロファイリング時にフレックスメッセージで判定し、介入戦略を自動決定する

---

## 設計思想

習慣化の成否を決める3つの問いがある：

1. **なぜ動くか**（動機の源泉） — そもそも何が自分を動かすのか
2. **なぜ止まるか**（挫折の構造） — 何が欠けると行動が止まるのか
3. **どう立て直すか**（回復の型） — 崩れた後にどうやって戻るか

この3軸をそれぞれ4分類し、4 × 4 × 4 = 64類型を定義する。
各類型に対して、AIコーチの介入戦略が1対1で決まる。

---

## 軸1: 動機の源泉（なぜ動くか）

Gretchen Rubinの「Four Tendencies」を基盤とし、習慣化の文脈に特化させた分類。

### 判定質問（フレックスメッセージ・2問）

**Q1:**
```
「毎日30分運動する」と自分で決めました。
2週間後、あなたの状態に一番近いのは？

[A] 決めたことだから続けている
[B] 運動の効果を調べて、納得できたから続けている
[C] ジムの予約や友人との約束があるから続いている
[D] 気分が乗る日だけやっている。「毎日」という縛りは外した
```

**Q2:**
```
上司から「資格を取れ」と言われました。
自分でもやや興味はあります。あなたの反応は？

[A] 言われなくても取るつもりだった。予定通り進める
[B] その資格が自分のキャリアに本当に必要か調べてから決める
[C] 上司に言われたなら頑張る。期限を決めてもらえると助かる
[D] 「取れ」と言われると逆にやる気が下がる。自分で選びたい
```

### 4分類

| コード | 名称 | 定義 | 動機の構造 |
|---|---|---|---|
| **M1** | 自律駆動型 | 自分で決めたことを自分で守れる | 内的期待 ○ / 外的期待 ○ |
| **M2** | 納得駆動型 | 理由と根拠が腑に落ちれば動く | 内的期待 ○（条件付き）/ 外的期待 △ |
| **M3** | 他者駆動型 | 誰かの期待・約束・報告先があると動く | 内的期待 × / 外的期待 ○ |
| **M4** | 自由駆動型 | 自分の選択としてやる。強制されると止まる | 内的期待 △ / 外的期待 × |

### 判定ロジック

```
Q1  Q2  → タイプ
A   A   → M1（自律駆動）
B   B   → M2（納得駆動）
C   C   → M3（他者駆動）
D   D   → M4（自由駆動）

不一致の場合:
A+B or B+A → M2寄りのM1（納得が入ると強い自律型）
A+C or C+A → M1寄りのM3（自律もできるが外圧で加速）
B+C or C+B → M3寄りのM2（理由があり外圧もあれば最強）
B+D or D+B → M2寄りのM4（自由だが理屈は好き）
C+D or D+C → M4寄りのM3（矛盾型 — 状況で切り替わる）
A+D or D+A → M1寄りのM4（自走するが縛られると反発）

※ 不一致の場合はプライマリ/セカンダリとして記録し、
   プライマリの戦略をベースにセカンダリの要素を加える
```

## 軸2: 挫折の構造（なぜ止まるか）

BJ Foggの行動モデル（B=MAP）を基盤に、「何が欠けると行動が止まるか」を4分類。

### 判定質問（フレックスメッセージ・2問）

**Q3:**
```
過去に続けようとして続かなかったこと、ありますか？
一番の原因に近いのは？

[A] 最初は燃えるけど、飽きて他のことに目移りした
[B] 忙しくなって、やる時間がなくなった
[C] 気づいたらやるのを忘れていた。思い出しても「まぁいいか」となった
[D] ちゃんとやれない日があると嫌になって、全部やめた
```

**Q4:**
```
新しい習慣を始めて1週間。今日はやる気が出ません。
一番近い心境は？

[A] 正直、もう別のことがやりたい
[B] 今日は本当に時間がない。物理的に無理
[C] あ、そういえば今日まだやってなかった
[D] 中途半端にやるくらいならやらない方がマシ
```

### 4分類

| コード | 名称 | Foggモデル上の位置 | 欠けるもの | 本質的な課題 |
|---|---|---|---|---|
| **F1** | 動機減衰型 | Motivationの低下 | 新鮮さ・刺激 | 同じことの繰り返しに耐えられない。常に新しい刺激を求める |
| **F2** | 能力不足型 | Abilityの不足 | 時間・余裕・環境 | やる意思はあるが、物理的条件が整わないと止まる |
| **F3** | きっかけ欠如型 | Promptの不在 | トリガー・リマインド | 思い出しさえすればやれるが、思い出せない |
| **F4** | 基準過剰型 | Abilityのハードル自己設定 | 完了の再定義 | 自分で設定した高い基準を満たせないと「やった」と認められない |

### 判定ロジック

```
Q3  Q4  → タイプ
A   A   → F1（動機減衰）
B   B   → F2（能力不足）
C   C   → F3（きっかけ欠如）
D   D   → F4（基準過剰）

不一致の場合:
A+C or C+A → F1寄りのF3（飽きやすく、忘れやすい = 最も離脱しやすい）
A+D or D+A → F1寄りのF4（新しいことに飛びつき、完璧にやろうとして燃え尽きる）
B+C or C+B → F2寄りのF3（忙しくて忘れる = 環境設計で解決可能）
B+D or D+B → F2寄りのF4（時間がない上に完璧主義 = 最低ラインの設定が最重要）
A+B or B+A → F1寄りのF2（飽きと忙しさの複合 = 短時間×変化で対応）
C+D or D+C → F3寄りのF4（忘れがちな上に完璧主義 = リマインド+基準緩和）

※ プライマリ/セカンダリで記録
```

## 軸3: 回復の型（どう立て直すか）

習慣が崩れた後の「立て直しパターン」。行動変容の維持期〜後退期の研究に基づく。

### 判定質問（フレックスメッセージ・2問）

**Q5:**
```
3日間サボってしまいました。
あなたが一番立ち直りやすいのは？

[A] 「もう一度やる」と自分で宣言し直す
[B] なぜサボったか分析して、仕組みを修正する
[C] 誰かに「また始めるよ」と報告する
[D] 小さいことから再開する。考えすぎない
```

**Q6:**
```
習慣を立て直すとき、一番助けになるのは？

[A] 明確な再スタート日を決めること
[B] サボった原因を特定して対策を立てること
[C] 一緒にやる人、見てくれる人がいること
[D] ハードルを極限まで下げて、とにかく1回やること
```

### 4分類

| コード | 名称 | 回復のメカニズム | コーチが提供すべきもの |
|---|---|---|---|
| **R1** | 宣言リセット型 | 「やる」と決め直すことで再起動する | 再スタートの儀式・節目の設定 |
| **R2** | 分析改善型 | 原因を理解し、仕組みを修正して再開する | 失敗分析のフレーム・データ |
| **R3** | 外部再接続型 | 他者との接続を回復させて再開する | チェックインの再開・報告先の設定 |
| **R4** | 最小行動型 | とにかく1つだけやって勢いを取り戻す | 最低ラインの提示・「1つだけ」の促し |

### 判定ロジック

```
Q5  Q6  → タイプ
A   A   → R1（宣言リセット）
B   B   → R2（分析改善）
C   C   → R3（外部再接続）
D   D   → R4（最小行動）

不一致の場合: Q5をプライマリ、Q6をセカンダリとして記録
（Q5は「実際の行動」、Q6は「理想の支援」を反映するため、Q5を優先）
```

## 64類型の構造

### 類型コード

`M{1-4}-F{1-4}-R{1-4}` の3文字コードで表現。

例:
- `M3-F1-R4` = 他者駆動 × 動機減衰 × 最小行動型
  → **大悟の推定類型**
  → 報告先があると動くが飽きやすい。崩れたら小さく再開
- `M1-F4-R2` = 自律駆動 × 基準過剰 × 分析改善型
  → 自分で動けるが完璧主義で燃え尽きる。原因分析で立て直す

### 64類型 一覧と介入戦略の要約

#### M1（自律駆動）× 各組み合わせ（16型）

| 類型 | 挫折 | 回復 | 介入の核 |
|---|---|---|---|
| M1-F1-R1 | 飽き | 宣言 | 定期的な目標更新の儀式を提供。データは最小限 |
| M1-F1-R2 | 飽き | 分析 | なぜ飽きるか構造を一緒に分析。変化の計画を立てる |
| M1-F1-R3 | 飽き | 外部 | 自走できるが飽き対策に他者を使う。ゆるい報告先を設定 |
| M1-F1-R4 | 飽き | 最小 | 飽きたら縮小版で維持。「0にしない」戦略 |
| M1-F2-R1 | 時間 | 宣言 | 時間ブロッキングの設計支援。宣言で固定 |
| M1-F2-R2 | 時間 | 分析 | 時間の使い方を分析。ボトルネック特定 |
| M1-F2-R3 | 時間 | 外部 | スケジュール共有で外圧を作る |
| M1-F2-R4 | 時間 | 最小 | 忙しい日は2分版を提示。時間ゼロでもできる形 |
| M1-F3-R1 | 忘れ | 宣言 | トリガー設定+宣言。「○○のあとにやる」を決め直す |

```
     ニックネームを教えてください。
     朝と夜、通知がほしい時間を教えてください。
     （例: 朝8時、夜22時）
  ↓
Bot: 設定完了しました。
     では、最初の習慣を追加しましょう。
     「追加 ○○」と送ってください。
```

### プロファイル結果の表示ルール

- 類型コード（M3-F1-R4等）はユーザーに見せない
- 日本語の平易な表現で特徴を伝える
- 「おすすめの関わり方」はタイプから自動算出するが、ユーザーが上書き可能
- 否定的な表現は避ける（「飽きやすい」→「新鮮さが大事」等）

### おすすめ関わり方の自動算出

```
M1（自律）→ デフォルト: 優しめ（干渉不要）
M2（納得）→ デフォルト: バランス（理由ベース）
M3（他者）→ デフォルト: バランス or しっかり（チェックイン重要）
M4（自由）→ デフォルト: 優しめ（圧をかけない）

F4（基準過剰）が入る場合 → 厳しめは非推奨（悪化する）
R3（外部再接続）が入る場合 → しっかりを推奨
```

## コーチの話し方スタイル

### 4スタイル定義

ユーザーが選択する「コーチのキャラクター」。関わり方（優しめ/バランス/厳しめ）とは独立した軸。

| コード | 名称 | 語尾・口調 | 想定ユーザー |
|---|---|---|---|
| `polite` | 丁寧 | です/ます調。穏やかで安心感がある | 初めて習慣化に取り組む人、圧が苦手な人 |
| `frank` | フランク | だよ/だね/じゃん。友達感覚 | 堅い言葉が苦手な人、気軽さを求める人 |
| `aniki` | 兄貴/姉御 | だ/だろ/しろ。体育会的な熱さ | 叱ってほしい人、背中を押されたい人 |
| `neutral` | 淡々 | ください/します。感情を排した事務的トーン | データ重視の人、感情的な関わりが不要な人 |

### 各スタイルのメッセージ例

#### 朝の問いかけ

| スタイル | メッセージ例 |
|---|---|
| 丁寧 | 「おはようございます、○○さん。今日のフォーカスを1つ教えてください」 |
| フランク | 「おはよう！今日は何やる？」 |
| 兄貴/姉御 | 「おはよう。今日の一発目を決めろ」 |
| 淡々 | 「おはようございます。本日のフォーカスを入力してください」 |

#### 達成時

| スタイル | メッセージ例 |
|---|---|
| 丁寧 | 「今週5/7達成です。着実に積み上がっていますね」 |
| フランク | 「5/7！いい感じじゃん」 |
| 兄貴/姉御 | 「5/7。悪くない。明日も取りに行け」 |
| 淡々 | 「今週の達成率: 71.4%（5/7）。先週比+14.3%」 |

#### 未達時（バランス関わり方の場合）

| スタイル | メッセージ例 |
|---|---|
| 丁寧 | 「今日は未達でした。何かありましたか？」 |
| フランク | 「今日はできなかったね。何があった？」 |
| 兄貴/姉御 | 「未達だ。理由を聞かせろ」 |
| 淡々 | 「本日の記録: 未達。原因の入力をお願いします」 |

#### 2日連続未達時（厳しめ関わり方の場合）

| スタイル | メッセージ例 |
|---|---|
| 丁寧 | 「2日連続の未達です。このまま3日目に入ると、ここまでの積み上げが崩れてしまいます。今日、最低ラインだけでも取り組みませんか？」 |
| フランク | 「2日連続サボってるよ。明日もやらなかったら終わるよ？今日1個だけやろう」 |
| 兄貴/姉御 | 「2日連続だ。言い訳はいらない。今すぐ最低ラインだけやれ。話はそれからだ」 |
| 淡々 | 「2日連続未達。統計上、3日連続で習慣継続率は大幅に低下します。最低ラインの実行を推奨します」 |

#### 危機介入時の回復促し

| スタイル | メッセージ例 |
|---|---|
| 丁寧 | 「少し間が空きましたね。完璧でなくても大丈夫です。今日、1つだけやってみませんか？」 |
| フランク | 「久しぶり！まぁそういう時もあるよ。1個だけやってみない？」 |
| 兄貴/姉御 | 「戻ってきたか。ブランクは気にするな。今日1つやれ。それだけでいい」 |
| 淡々 | 「前回記録から○日経過。再開する場合、最小の1項目から始めることを推奨します」 |

### 話し方 × 関わり方のマトリクス

話し方（4種）と関わり方（3種）は独立した軸。組み合わせは 4 × 3 = 12パターン。

```
話し方: コーチの「キャラクター」（語尾・トーン・温度感）
関わり方: コーチの「介入の強さ」（頻度・厳しさ・追及度）

例:
- フランク × 優しめ → 友達感覚で、できたことにフォーカス
- フランク × 厳しめ → 友達だけど容赦ない。「それサボりじゃん」
- 兄貴 × 優しめ → 熱いが追い込まない。「できた時だけ声かけるぞ」
- 淡々 × 厳しめ → 感情なしでデータで追い込む。最も冷徹
```

### 話し方の変更

- 「話し方変更」とLINEで送信 → Step 5のカルーセルを再送信
- リッチメニューの「設定」からもアクセス可能
- 変更は即時反映。過去のメッセージには影響しない

## リセット機能

### 3段階のリセット

| コマンド | 対象 | 確認メッセージ |
|---|---|---|
| 「リセット」 | 全データ（プロファイル+習慣+記録） | 「すべてのデータをリセットします。プロファイリングからやり直しになります。よろしいですか？」 |
| 「プロファイルリセット」 | プロファイルのみ（習慣・記録は保持） | 「プロファイルを再設定します。習慣と記録はそのまま残ります。よろしいですか？」 |
| 「関わり方変更」 | 関わり方の設定のみ | Step 4のフレックスメッセージを再送信 |
| 「話し方変更」 | 話し方スタイルのみ | Step 5のカルーセルを再送信 |

- リッチメニューの「設定」からも全リセットにアクセス可能
- プロファイルリセット後は質問から再開
- 関わり方変更・話し方変更は質問なしで即変更

## DB設計への反映

### user_profiles テーブル（最終版）

```sql
create table user_profiles (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references users(id) on delete cascade,
  nickname text,
  
  -- 軸1: 動機の源泉
  motivation_type text check (motivation_type in ('M1', 'M2', 'M3', 'M4')),
  motivation_secondary text, -- 不一致時のサブタイプ
  motivation_q1 char(1), -- 生回答
  motivation_q2 char(1),
  
  -- 軸2: 挫折の構造  
  failure_type text check (failure_type in ('F1', 'F2', 'F3', 'F4')),
  failure_secondary text,
  failure_q3 char(1),
  failure_q4 char(1),
  
  -- 軸3: 回復の型
  recovery_type text check (recovery_type in ('R1', 'R2', 'R3', 'R4')),
  recovery_secondary text,
  recovery_q5 char(1),
  recovery_q6 char(1),
  
  -- 類型コード（自動生成）
  type_code text generated always as (
    motivation_type || '-' || failure_type || '-' || recovery_type
  ) stored,
  
  -- 関わり方（自動算出 + ユーザー上書き可）
  coach_style text check (coach_style in ('gentle', 'balanced', 'strict')) default 'balanced',
  coach_style_auto text, -- 自動算出値（上書き前の値を保持）
  
  -- 話し方スタイル
  speech_style text check (speech_style in ('polite', 'frank', 'aniki', 'neutral')) default 'polite',
  
  -- 通知設定
  life_rhythm text check (life_rhythm in ('morning', 'night', 'irregular')),
  morning_notify_time time default '08:00',
  evening_notify_time time default '22:00',
  
  -- 成功体験（任意）
  success_experience text,
  
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- インデックス
create index idx_user_profiles_type_code on user_profiles(type_code);
```
