# 習慣管理LINE Bot — 昌平君への引継ぎ

> 起案: 河了貂（2026-04-14）
> 担当: 昌平君
> 目的: 大悟の習慣化を構造的に支援するLINE Botシステム

---

## 背景

大悟の習慣化が続かない根本原因は2つ：
1. **外部プレッシャーがないと動けない**（一人で完結する習慣は持たない）
2. **ENFP特性で同じことの繰り返しに耐えられない**（着想が次に飛ぶ）

これを踏まえ、LINE Botが「習慣化のプロ」として介入する仕組みを作る。
3ヶ月自分で使い倒し、プロダクト化判断は7月以降。

---

## プロダクト方針

- 入室LINE通知とは**完全に別システム**（別リポジトリ・別LINE Bot）
- 将来プロダクト化の余地を残す設計（マルチユーザー前提のDB設計）
- 最小構成で動くものを早く作り、使いながら磨く

---

## インフラ構成

| レイヤー | 技術 |
|---|---|
| Bot | LINE Messaging API（新規チャネル） |
| バックエンド | Cloudflare Workers + Hono（入室LINE通知と同構成で可）or Vercel |

## AI人格：習慣化コーチ

単なるリマインダーではなく、**習慣化の専門知識を持つAIコーチ**として振る舞う。

### 知識ベース
- James Clear「Atomic Habits」の原則（Habit Stacking、2分ルール、4つの法則）
- 行動科学（トリガー設計、報酬設計、環境設計）
- KPT振り返り手法
- 習慣化の科学的知見（18-200日で定着、7週間が挫折ポイント）

### 大悟専用のコンテキスト（初期設定）
- ENFP-A：着想暴走・新しいものへの飛びつきを検知して指摘
- 外部プレッシャー型：「報告する相手がいる」構造が動機になる
- 完璧主義の罠：「今日は3行でいい」と下限を示す
- 数字で語る：「今週5/7達成」のように事実ベースで伝える

### 介入スタイル
- 序盤（1-3週）: 励まし寄り。「やれただけで十分だ」
- 中盤（4-7週）: 事実ベース。「3日連続で未達だ。何が障害になっている？」
- 定着期（8週〜）: 厳しめ。「2日サボった。明日もやらなければ習慣は死ぬ。今すぐ1つだけやれ」
- ただし、常に**問いを投げる**スタイル。命令ではなく気づかせる

### 口調
- プロダクト化を見据え、河了貂とは別の人格にする
- 落ち着いた敬語ベースだが、核心を突く（コーチ的）
- 例：「今日は1つだけ聞かせてください。何ができましたか？」
- 例：「3日連続の未達です。習慣の難易度を下げましょうか、それとも項目を減らしましょうか？」

## コア機能

### 0. 初回プロファイリング（友だち追加 or リセット時）

友だち追加した瞬間、またはユーザーが「リセット」と送信した時に起動。
**フレックスメッセージ**でステップ形式の質問を出し、ユーザーのプロファイルを構築する。
**詳細は `profiling-64types.md` を参照。**

#### フロー

```
[友だち追加 or 「リセット」送信]
  ↓
Bot: はじめまして。あなたの習慣づくりを一緒に進めるコーチです。
     まず、あなたのことを教えてください。（2分で終わります）
  ↓
【Step 1】フレックスメッセージ: 基本情報
  ├─ 名前（ニックネーム可）
  └─ 生活リズム（朝型 / 夜型 / 不規則）→ ボタン選択
  ↓
【Step 2】フレックスメッセージ: 習慣化の傾向
  ├─ 過去に習慣化に成功したことは？（自由入力）
  ├─ 習慣が続かなくなる原因は？→ カルーセルで選択
  │   [飽きる] [忙しくなる] [忘れる] [完璧にやろうとする] [一人だと続かない] [その他]
  └─ 複数選択可
  ↓
【Step 3】フレックスメッセージ: 関わり方の設定
  ├─ コーチの関わり方を選んでください → カルーセル
  │   [優しめ] 励まし中心。「できたこと」にフォーカス
  │   [バランス] 事実ベース。良い点も課題も伝える
  │   [厳しめ] 容赦なし。サボりには即指摘
  ├─ リマインドの頻度 → ボタン
  │   [最小限] 2日未達時のみ
  │   [普通]  当日未達 + 2日未達
  │   [しっかり] 朝の確認 + 当日未達 + 2日未達
  └─ 通知時間の設定（朝: ○時 / 夜: ○時）
  ↓
【Step 5】フレックスメッセージ: コーチの話し方
  ├─ カルーセルで4スタイルを提示（各スタイルのサンプルメッセージ付き）
  │   [丁寧] です/ます調。穏やかで安心感
  │   [フランク] だよ/だね。友達感覚
  │   [兄貴/姉御] だ/だろ/しろ。体育会的な熱さ
  │   [淡々] ください/します。感情を排した事務的トーン
  └─ 単一選択
  ↓
Bot: プロファイル設定が完了しました。
     [プロファイル概要をフレックスメッセージで表示]
     いつでも「リセット」「話し方変更」「関わり方変更」で再設定できます。
     では、最初の習慣を追加しましょう。「追加 ○○」と送ってください。
```

#### プロファイリングで取得する情報

| 項目 | 取得方法 | 用途 |
|---|---|---|
| ニックネーム | テキスト入力 | 呼びかけに使用 |
| 生活リズム | ボタン選択 | デフォルトの通知時刻設定 |
| 習慣化の成功体験 | テキスト入力 | AIコーチの介入時の参照 |
| 続かない原因 | カルーセル複数選択 | 介入パターンのカスタマイズ |
| 関わり方 | カルーセル単一選択 | コーチの口調・厳しさの基準 |
| リマインド頻度 | ボタン選択 | cron設定に反映 |
| 通知時間 | テキスト入力 | 朝・夜の通知時刻 |

#### リセット機能

- 「リセット」と送信 → 確認メッセージ（「プロファイルと習慣データをすべてリセットしますか？」）
- 「プロファイルリセット」→ プロファイルのみ再設定（習慣データは保持）
- 「習慣リセット」→ 習慣項目のみクリア（プロファイルは保持）
- リッチメニューの「設定」からもアクセス可能

### 1. 習慣項目の管理（LINE上で完結）

```
ユーザー: 「追加 朝散歩」
Bot: 「朝散歩」を追加しました。達成ラインと最低ラインを教えてください。
ユーザー: 「達成: 30分散歩 / 最低: 外に出る」
Bot: 設定しました。
    達成（黒塗り）: 30分散歩
    最低（斜線）: 外に出る

ユーザー: 「削除 朝散歩」
ユーザー: 「一覧」→ 現在の習慣項目を表示
```

- 各習慣に「達成ライン（黒塗り）」と「最低ライン（斜線）」の2段階を設定
- 習慣を「行動習慣」「生活習慣」にカテゴリ分け可能

### 2. 朝の問いかけ（毎朝 設定時刻）

```
Bot: おはようございます。今日のフォーカスを1つ教えてください。
ユーザー: Elencoのβテスター招待文を送る
Bot: 了解しました。今日のフォーカス：βテスター招待文の送付
    今日の習慣：[朝散歩] [Elenco作業] [読書]
```

### 3. 夜の記録（毎晩 設定時刻）

```
Bot: 今日の記録をつけましょう。

【習慣チェック】該当するものを番号で送ってください（達成=番号、最低限=番号m）
1. 朝散歩  2. Elenco作業  3. 読書  4. 運動

【ハイライト】今日の1行

【振り返り（書けるものだけ）】
K(できたこと):
P(課題):
T(次やること):
```

- 返信は自由形式でもパース可能にする（厳密なフォーマット強制はしない）
- 「1,2,3m」→ 1,2は達成、3は最低ライン達成、4は未達

### 4. リマインド（段階的）

| 条件 | 時刻 | メッセージ例 |
|---|---|---|
| 当日未達の習慣あり | 20:00 | 「まだ[朝散歩]が未達です。最低ラインでもOKです」 |
| 夜の記録が未入力 | 23:00 | 「今日の記録がまだです。1行だけでも送ってください」 |
| 2日連続未達（序盤） | 翌朝 | 「2日空きました。今日は最低ラインだけ狙いましょう」 |
| 2日連続未達（定着期） | 翌朝 | 「2日連続の未達です。このまま3日目に入ると習慣はリセットされます。今日、1つだけやってください」 |

### 5. 可視化Webページ（リッチメニューからリンク）

#### 日次ビュー
- 日付ごとのハイライト・KPT・習慣達成状況
- カレンダー形式で一覧

#### 月次トラッカー
- 縦軸：日付（1-31日）、横軸：習慣項目
- 黒塗り（達成）/ 斜線（最低ライン）/ 空白（未達）
- 行動習慣・生活習慣で色分け
- 月次達成率の自動計算

#### 月次振り返りページ
- 今月の目標（月初に設定）と達成状況
- 習慣ごとの達成率推移
- 月末に振り返り用の余白

### 6. リッチメニュー構成

```
┌──────────┬──────────┬──────────┐
│  今日の   │  習慣    │  設定    │
│  記録     │  一覧    │          │
├──────────┼──────────┼──────────┤
│  日次     │  月次    │  月の    │
│  ログ     │ トラッカー│  目標    │
└──────────┴──────────┴──────────┘
```

## DB設計（Supabase / 概要）

```
users
  id, line_user_id, display_name, coach_phase (序盤/中盤/定着期), 
  onboarding_completed (boolean), created_at

user_profiles
  id, user_id, nickname, life_rhythm (朝型/夜型/不規則),
  success_experience (text), failure_reasons (jsonb: [飽きる, 忘れる, ...]),
  coach_style (優しめ/バランス/厳しめ), reminder_frequency (最小限/普通/しっかり),
  morning_notify_time, evening_notify_time, created_at, updated_at

habits
  id, user_id, name, category (行動/生活), 
  achievement_line, minimum_line, sort_order, is_active, created_at

daily_logs
  id, user_id, date, focus (朝のフォーカス), 
  highlight (1行ハイライト), 
  kpt_keep, kpt_problem, kpt_try, created_at

habit_records
  id, user_id, habit_id, date, 
  status (achieved / minimum / missed), created_at

monthly_goals
  id, user_id, year_month, goals (jsonb), reflection (月末振り返り), created_at

reminder_logs
  id, user_id, type, sent_at
```
